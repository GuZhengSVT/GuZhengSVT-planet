<h2 id="缘由">缘由</h2>
<p><a href="https://elm-lang.org/" target="_blank" rel="external nofollow noopener noreferrer">Elm</a> 是一种函数式编程语言，这个项目在2012年才开始，对编程语言界这群动辄上个世纪的老东西们来说，挺新的。这个语言承诺不会出现运行时异常，并且声称有很出色的性能和“令人愉快的开发人员体验”（虽然每个语言都这么说，但是个人感觉着实没几门语言在大型项目里写起来真的令人愉快）。</p>
<p>实际上我也没有看见过有什么网站/开源项目是使用Elm写的——至少比较有名的开源项目里没怎么见过，但是 Elm Structure 却很经常出现在各种新兴的 UI 框架里，无论是 Web 端还是 Native GUI，比如 <a href="https://redux.js.org/" target="_blank" rel="external nofollow noopener noreferrer">Redux</a>，<a href="https://github.com/iced-rs/iced" target="_blank" rel="external nofollow noopener noreferrer">iced-rs</a>，<a href="https://github.com/Relm4/Relm4" target="_blank" rel="external nofollow noopener noreferrer">Relm4</a>，<a href="https://raphlinus.github.io/rust/gui/2022/05/07/ui-architecture.html" target="_blank" rel="external nofollow noopener noreferrer">Xilem</a>…等等等。</p>
<h2 id="the-elm-architecture">The Elm Architecture</h2>
<p>Elm 是一种用于构建交互式程序的模式。</p>
<p>按照他们自己的介绍，这种架构是写着写着自己变成这样的：</p>
<blockquote>
<p>This architecture seems to emerge naturally in Elm. Rather than someone inventing it, early Elm programmers kept discovering the same basic patterns in their code. It was kind of spooky to see people ending up with well-architected code without planning ahead!</p>
</blockquote>
<h3 id="basic-pattern">Basic Pattern</h3>
<p>在 Elm 的 UI 模型中，人机交互主要分为了三个部分：</p>
<ul>
<li>Model：用来表示应用程序的状态；</li>
<li>View：一种将状态转换为用户界面的方法；</li>
<li>Update：一种根据消息更新状态的方法；</li>
</ul>
<p>因此整个消息循环实际上呈现出下面这样的结构：</p>
<div class="highlight" id="id-1"><pre tabindex="0" class="chroma"><code class="language-plaintext" data-lang="plaintext"><span class="line"><span class="cl">    +-----&gt; View -------+
</span></span><span class="line"><span class="cl">    |                   |
</span></span><span class="line"><span class="cl">    |                   v
</span></span><span class="line"><span class="cl">  Model &lt;----------- Update</span></span></code></pre></div><p>看起来有些像以前进化过的 <a href="https://developer.mozilla.org/en-US/docs/Glossary/MVC" target="_blank" rel="external nofollow noopener noreferrer">MVC(Model-View-Controller)</a> 模型？但实际上不太一样，Elm 的架构要更加简单一些。在传统 MVC 架构中，View 的行为是通过 Controller 直接控制的，但是 View 的更改会直接影响 Model，然后 Controller 通过监听 Model 的变化来进行响应（也有的 MVC 模型是 Controller -&gt; Model -&gt; View 的，然后 View 与 Controller 进行双向交互响应）；而 Elm 的架构中，View 操作会直接调用 Update 方法，Update 方法会采取措施更新 Model，最终 View 根据 Model 重新构建出用户界面达到一次响应。</p>
<p>我们可以发现 MVC 和 Elm 架构最大的区别是 Elm 保证了数据的单向流动。</p>

<h2 id="基本单元">基本单元</h2>
<p>在 Elm Structure 中，<strong>组件</strong> 是整套UI系统的基本单元。一个 <strong>组件</strong> 被描述为构造用户界面的基本块，一个组件可以由多个组件组合而成。</p>
<h3 id="状态和消息">状态和消息</h3>
<p>每个组件都有以下三种数据结构：</p>
<ol>
<li><code>Model</code>：组件的状态；</li>
<li><code>Input</code>：组件从外部接收的消息；</li>
<li><code>Output</code>：组件向外部发送的消息；</li>
</ol>
<p>其中 <code>Model</code> 一般是一个结构体，里面存储着组件的持久化状态信息，而 <code>Input</code> 和 <code>Output</code> 是枚举（Enum）类型，用来指示消息类型，并携带消息的内容。</p>
<p>在 Relm4 的设计中，组件依靠 <code>init</code> 和 <code>update</code> 函数进行消息的处理。当组件接收到消息时，<code>update</code> 函数就会被调用，然后函数根据枚举类型进行消息的分发处理，例如更新 <code>Model</code>，将消息转码之后转发 <code>Output</code> 等等。</p>
<h3 id="组件">组件</h3>
<p>组件是状态、消息、处理逻辑和界面描述的组合，一个组件就表示了一个完整的具有响应性和对应行为的用户界面，例如一个会发送点击信号的按钮、能以编程方式和用户点击方式更改选中状态的单选框等等。</p>
<p>在 Relm4 中，组件是通过 <code>SimpleComponent trait</code> 来定义的，这个 <code>trait</code> 会要求开发者实现 <code>Model</code>、<code>Input</code>、<code>Output</code> 的定义以及 <code>init</code> 和 <code>update</code> 函数，并提供了 <code>view!</code> 宏用来生成用户界面。</p>
